# QUBOOK サービス要件定義書

## 1. サービス概要
親から子へ送る「子の誕生を記念する絵本」
子供の名前とその由来から生み出される世界に唯一の物語
量子アニーリングとAIを活用し、質の高い絵本を数分で生成
### 1.1 背景
- 生まれた子供に送るものとして、「絵本」が注目されている
- 子供への思い、名前、名前の由来を込めたオリジナル絵本の需要はありそう
- 既存のパーソナライズ絵本サービスは、名前の文字を物語に組み込む表面的なカスタマイズに留まっている
(オリジナルストーリーといいつつ、登場人物の名前が変わっているだけなど)
- 従来の手作業による制作プロセスは時間と人的コストがかかる

### 1.2 市場分析・競合調査 (競合サービスの調査 → 比較して、どこが優位なのかという表まで作成したい)
#### 競合サービス分析（Wonderbly）
- 基本機能
  - 名前のスペルを物語に組み込む基本的なカスタマイズ
  - キャラクターの外見選択機能
  - パーソナルメッセージや写真の追加機能

#### 競争優位性
- 日本語・日本文化に特化したストーリー展開
- 名前の由来、親から子へのメッセージに基づくオリジナリティな物語・絵の実現
- 量子アニーリングとAIの統合による高速コンテンツ生成

### 1.3 目的
1. 名前に込められた想いの世代間伝承
2. 親から子への想いのストーリー化
3. 命名時のエピソードの視覚的表現
4. 家族の歴史や願いの継承
5. 日本文化要素を活かした独自の世界観構築

### 1.4 製品仕様要件
#### 1.4.1 絵本の基本仕様
- ページ数：20-24ページ（表紙、裏表紙含む）
- サイズ：A4横型（297mm × 210mm）
- 製本：上質紙、ハードカバー仕様

#### 1.4.2 ページレイアウト
- 文字数：1ページあたり100-150文字
- 文字サイズ：本文14-16pt
- イラスト配置：ページ面積の60-70%
- 余白：上下左右20mm以上確保

#### 1.4.3 コンテンツ仕様
- ひらがなベースの読みやすいストーリー
- 日本的要素の効果的な導入
  - 干支（12種）
  - 自然現象（太陽、月、虹、星）
  - 気象要素（雲、雨、雪、風）
  - 日本の象徴的な植物（桜、竹、菊、ひまわり、梅、モミジ）
- 量子アニーリングによる物語構造の最適化
- AI生成イラスト（解像度300dpi以上）
- 文字別ストーリー展開（各文字2ページ程度）
- 起承転結による4部構成

#### 1.4.4 システム提供形態
- Webアプリケーション形式
- レスポンシブデザイン対応
- 対応ブラウザ：Chrome, Safari, Firefox, Edge

## 2. 機能要件

### 2.1 必須機能（MVP）
#### 2.1.1 ユーザー入力機能
- 子供の基本情報登録
  - 名前（ひらがな）
  - 性別
  - 生年月日
- アバター選択
- 名前の由来情報
  - 命名の理由
  - 込められた願い
  - 特別なエピソード

#### 2.1.2 コンテンツ生成機能
- 量子アニーリングによるストーリー最適化
- AI文章生成エンジン連携
- AIイラスト生成処理
- レイアウト自動最適化

#### 2.1.3 プレビュー・編集機能
- ページ単位のプレビュー表示
- ストーリーフロー確認機能
- 文章・イラスト編集機能

### 2.2 利用フロー
1. 基本情報入力
   - 名前入力
   - 個人情報登録
   - アバター選択
2. 名前の由来情報入力
   - ストーリー記述
   - 願いの入力
   - エピソード登録
3. カスタマイズ設定
   - ストーリー選択
   - キャラクター設定
   - 背景選択
4. 確認・編集
   - プレビュー確認
   - 内容編集
   - 最終確認
